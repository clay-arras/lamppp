## v0 locks

- **Fusible set = same-shape binary elementwise only.** Unary ops and
  broadcast/non-contiguous ops are boundaries.
- **`float` only.** Codegen hardcodes `float`; mixed-dtype and casts are out.
- **Single output = the group root.** No escape analysis, no multi-output.
- **No cache canonicalization.** Naming leans on the walk's post-order.

## Decision — `LMP_ENABLE_FUSION` + single-PR workflow

A fusion group cannot execute until Part 3 exists (no compile/launch). So:

- Introduce a **CMake/compile flag `LMP_ENABLE_FUSION`**, **off by default**.
  Flag off → today's eager-via-capture path (#67); CUDA CI stays green. Flag
  on → the fusion pipeline (Parts 2+3) is reachable.
- Through Part 2, even with the flag *on*, `realize()`'s **value path stays
  the eager arm**. The `FusedGraph` + codegen pipeline is built and exercised
  through a **test/inspection entry point** (build a graph, assert
  membership + golden source) — it does not yet drive results. Part 3 wires
  `evaluate()` into `realize()` under the same flag.
- Because the feature is only meaningfully testable **end-to-end** (codegen
  is worthless until it compiles and launches), Parts 2 and 3 develop on
  **one branch / one PR**, behind the flag, and merge when the flag-on path
  passes end-to-end on CUDA CI.

## Item 1 — Partition walk + `FusedGraph` membership

**What.** Add the fusible branch to `NVRTCInductorBackend::realize` (today
it is only the boundary arm) and implement `FusedGraph` extraction via the
`realize`/`visit` mutual recursion. Membership sets: `inputs` (group leaves),
`interior` (fused ops), `outputs` (root only), `seen` (DAG memo), with
`slot` dedup. `shape_ok(n, g)` rejects a node whose shape differs from the
group's single shape (v0 is same-shape).

```cpp
// inductor — the per-group artifact (the "form")
struct FusedGraph {
  std::vector<const tensor::TensorImpl*>        inputs;    // group leaves → in0,in1,…
  std::vector<tensor::TensorImpl*>              outputs;   // root only (v0)
  std::unordered_set<const tensor::TensorImpl*> interior;  // fused ops (unordered)
  std::unordered_set<const tensor::TensorImpl*> seen;      // DAG memo
  std::unordered_map<const tensor::TensorImpl*, int> slot; // dedup → index into inputs

  void add_input(const tensor::TensorImpl* n) {            // plain-map dedup
    if (!slot.count(n)) { slot[n] = (int)inputs.size(); inputs.push_back(n); }
  }

  // filled by Item 2
  std::vector<const tensor::TensorImpl*>                      topo;
  std::unordered_map<const tensor::TensorImpl*, std::string>  name;
  void topo_sort();
  std::string codegen() const;   // Item 3 → source string
};
```

```cpp
// the mutual recursion — visit decides membership, realize forces boundaries
void NVRTCInductorBackend::visit(tensor::TensorImpl* n, FusedGraph& g,
                                 bool is_root) {
  if (g.seen.count(n)) return;                       // DAG memo
  g.seen.insert(n);
  if (!n->is_deferred()) { g.add_input(n); return; } // realized leaf → input
  if (!n->lazy_op()->is_fusible() || !shape_ok(n, g)) {
    realize(n);                                      // force boundary FIRST…
    g.add_input(n);                                  // …now realized → input
    return;
  }
  for (auto& op : n->lazy_op()->inputs) visit(op.get(), g);   // operands first
  g.interior.insert(n);
  if (is_root) g.outputs.push_back(n);               // single output (v0)
}
```

The load-bearing invariant: every boundary operand is `realize`d *during* the
walk (the `realize(n)` call above), so when the DFS unwinds, all of a group's
leaves are already materialized. `shape_ok` compares against the group's
shape, which is fixed by the root (`outputs[0]->shape()`); any deferred node
whose shape differs is treated as a boundary and realized through its own
path.

**Value path during Part 2.** `realize()` keeps the #67 eager arm as the
source of truth; `visit`/`FusedGraph` are reached from a test/inspection
entry point (e.g. `build_fused_graph(impl)`), not yet wired to drive output.

**Why.** Establishes correct *grouping* — inputs vs interior vs output —
independent of codegen. The trickiest behavior (boundaries realized first,
DAG memo preventing double-visit, dedup assigning stable input slots) is
proven here on real graphs before any source is generated.

**Done when.** On #66's graph-equivalence fixtures, membership matches
hand-derived expectations: boundaries are realized and enter as deduped
inputs; a diamond (shared operand) is visited once and produces one input
slot; `slot` indices are stable and contiguous.

## Item 2 — `topo_sort` + SSA naming

**What.** Order the **interior op nodes** into `topo` and assign every node a
kernel-local name. Inputs are **not** ordered — they are kernel parameters,
enumerated in `slot` order. Two namespaces:

- **`v{i}`** — an **input load var**: the value read once from global memory,
  `float v{i} = in{i}[idx];`. Indexed by `slot` (v0, v1, …).
- **`t{k}`** — an **interior compute var**: an op result held in a register,
  `float t{k} = <expr>;`. Indexed by `topo` position (t0, t1, …).

`topo_sort` leans on the walk's post-order — `visit` already recurses into
operands before inserting a node, so discovery order is a valid topological
order. No canonical re-sort (caching is Part 3's concern).

```cpp
void FusedGraph::topo_sort() {
  // `topo` is filled from the post-order visit recorded in Item 1
  for (int i = 0; i < (int)inputs.size(); ++i)
    name[inputs[i]] = "v" + std::to_string(i);
  for (int k = 0; k < (int)topo.size(); ++k)
    name[topo[k]] = "t" + std::to_string(k);
}
```

**Why this is the seam.** Codegen (Item 3) emits one line per interior node,
`t{k} = expr(operands)`, and resolves each operand by `name` lookup — an
input resolves to its `v{slot}`, an interior operand to its `t{k}`.
Topological order is exactly the guarantee that every operand name is already
defined when a node is emitted; without it, a node could reference a `t` that
hasn't been computed. Keeping order + naming in their own method lets a future
canonicalizing strategy replace it without touching codegen.

**Done when.** For representative graphs, `topo` is a valid order (every
node's operands precede it) and `name` covers all inputs and interior nodes
deterministically.

## Item 3 — Codegen → source string + arg layout

**What.** Assemble a single CUDA kernel source from the ordered `FusedGraph`,
plus the `[inputs…, outputs…]` arg layout that Part 3's launch will gather
pointers against. Structure:

- **params** — `const float* __restrict__ in{i}` per input, then
  `float* __restrict__ out{o}` per output, then `int n`.
- **loads** — `float v{i} = in{i}[idx];` per input.
- **body** — flat walk over `topo`; each node emits
  `float t{k} = <codegen_expr() with {0}/{1} replaced by operand names>;`.
- **stores** — `out{o}[idx] = {name[output]};` per output.
- **wrapper** — `extern "C" __global__` + grid-stride loop (stable symbol for
  `cuModuleGetFunction`; launch config decoupled from `n`).

Per-op RHS templates are already the per-subclass `codegen_expr()` virtual —
no central `OpKind` map. Codegen reads `node->lazy_op()->codegen_expr()` and
substitutes `name[operand]` into each positional placeholder. **`float` is
hardcoded** in v0.

For `out = (a + b) * c` (all same-shape, all `float`):

```cuda
extern "C" __global__
void fused(const float* __restrict__ in0,   // a  → v0
           const float* __restrict__ in1,   // b  → v1
           const float* __restrict__ in2,   // c  → v2
           float* __restrict__ out0,        // root
           int n) {
  for (int idx = blockIdx.x * blockDim.x + threadIdx.x;
       idx < n; idx += gridDim.x * blockDim.x) {
    float v0 = in0[idx], v1 = in1[idx], v2 = in2[idx];   // loads, slot order
    float t0 = v0 + v1;    // interior[0]  AddFn  "{0} + {1}"
    float t1 = t0 * v2;    // interior[1]  MulFn  "{0} * {1}"  (t0 defined ✓)
    out0[idx] = t1;        // store
  }
}
```

Arg layout for this kernel: `[in0=a, in1=b, in2=c, out0=root]` — the exact
order Part 3 gathers device pointers in. Param-emit order and arg-gather order
both derive from the same `inputs`/`outputs` vectors, so they cannot drift.

**Why.** Turns a grouped + ordered graph into compilable source. Flat emit
over `topo` means no recursion and no operator-precedence/paren handling —
the post-order already linearized the expression tree into SSA statements.

**Done when.** Generated source for representative same-shape `float` graphs
is structurally correct under a **golden-source test**: params/loads/body/
stores in `inputs` then `outputs` order, operand names substituted correctly
into every `codegen_expr()`, and the emitted arg layout matches param order.
(Actually compiling/launching is Part 3.)

## Sequencing

```
Item 1 ─▶ Item 2 ─▶ Item 3 ─▶ (Part 3: evaluate = compile + cache + launch)
membership   order      source string + arg layout
```

Linear — each item consumes the prior's output. Item 1 is the bulk (the
partition logic); Items 2–3 are mechanical given correct membership. All
three land behind `LMP_ENABLE_FUSION` on the shared Part 2+3 branch.

## Future extensions (explicitly deferred)

- **Multi-dtype + casts.** dtype is already captured (`TensorImpl::type()`,
  `OpMeta::dtype`) but unused in v0. Generalizing codegen off `float` means a
  per-tensor type in params/loads/temps and a per-op cast rule (dtypes need
  not match across operands).
- **Automatic broadcasting.** Broadcast/non-contiguous ops are boundaries in
  v0 (flat same-shape indexing only). Fusing them needs shaped/strided
  indexing in the generated loop.
- **Unary ops.** No unary `LazyFunction` exists yet; unary nodes are
  boundaries in v0. Adding `ElementwiseUnaryFn` subclasses with their own
  `codegen_expr()` (e.g. `Relu → "fmaxf(0.0f, {0})"`, `Exp → "expf({0})"`)
  folds them into groups with no change to the walk.
- **Escape analysis / multi-output.** v0 emits a single output (the root). A
  later pass detects interior nodes whose value is live outside the group
  (liveness signals only) and adds extra stores → multi-output kernels.
- **Module-cache canonicalization.** `topo_sort` is the seam: replace
  post-order naming with a structure-derived canonical order so structurally
  identical graphs emit byte-identical source → cache hits. Pairs with
  Part 3's hash → module cache.
- **Cross-`.so` static-registration survival (carried over from Part 1.2 §6).**
  The lazy-backend registrar (`LMP_REGISTER_LAZY_BACKEND`, shipped #67) uses
  the existing static-initializer style. If `inductor` becomes a separate
  shared lib whose symbols nothing in `tensor_core` references, the linker can
  drop that TU and the backend never registers. Robust survival
  (`--whole-archive` / `-u` / an explicit init call) was deferred in Part 1
  and remains deferred — flagged here so it isn't lost.
